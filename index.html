Core Features

What's under the hood

Every feature maps to real SQL queries and Python logic — this is where the actual work lives.

📋

Report Card System

Tracks 4 exam types — Weekly Test 1 & 2, Half Yearly, and Final Exam — across 5 subjects (Physics, Chemistry, Math, English, CS). Calculates total, percentage, and auto-assigns grades A+ through F.

📈

Progress Analytics

Side-by-side bar charts built with Matplotlib showing a student's performance across all 4 exams per subject. Includes a dashed pass-mark line at 45% for quick visual reference.

💳

Fee Management

Quarterly fee tracking (Jan–Mar, Apr–Jun, Jul–Sep, Oct–Dec) at ₹7,500 per quarter. Supports marking payments, late-payment alerts, and full CRUD on fee records per student.

💰

Salary Module

Calculates net in-hand salary from basic pay + allowances − 8% tax − leave deductions. Admin can update allowances and leave count; net salary recalculates automatically.

📝

Assignment Tracker

Subject-wise assignment records with due date and submission status (Pending / Submitted). Teachers can add, update, or delete assignments; students view their list.

🔐

Authentication & CRUD

Login validated against the MySQL users table. Full Create / Read / Update / Delete operations on every entity — students, teachers, marks, fees and salary — via raw SQL and mysql-connector-python.

Tech Stack

Built with

The core project is Python + MySQL. Streamlit was added as a frontend layer on top of the existing logic.

🐍 Backend — Python 3.6+

All business logic written in Python. Database operations use mysql-connector-python to run raw SQL. Data manipulation with pandas DataFrames. Charts via matplotlib and numpy.

🗃️ Database — MySQL

Relational schema with 6 tables linked via foreign keys. All CRUD written as raw SQL — INSERT SELECT UPDATE DELETE — passed through cursor.execute(). No ORM used.

🖥️ Frontend — Streamlit

Wraps Python logic in a web UI. Uses st.tabs st.columns st.dataframe st.pyplot. Custom CSS injected via st.markdown(unsafe_allow_html=True) for the dark theme.

🔐 Auth & Session

Login validated against MySQL users table. Role stored in st.session_state and used to route to the correct portal. Password update writes back to the database.

Known Limitations
CLI

In the original Python CLI version, the password input is not masked — it was typed as plain text in the terminal.

Demo

The Streamlit version uses in-memory Pandas demo data instead of a live MySQL connection, so changes reset on page reload.

Scale

Designed for a single class (Class XII). Multi-class or multi-school support would require schema changes and additional role logic.
